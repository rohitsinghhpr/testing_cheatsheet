cart functinality test cases

1. Adding Items to Cart

  1. Verify adding a single item to the cart.
  2. Verify adding multiple items to the cart.
  3. Verify adding the same item multiple times updates the quantity.
  4. Verify adding out-of-stock items to the cart (should not be allowed).
  5. Verify adding items with special discounts or offers to the cart.
  6. Verify adding items with variations (e.g., size, color) to the cart.
  7. Verify the maximum quantity limit for adding items to the cart.
  8. Verify adding digital/downloadable products to the cart.

2. Removing Items from Cart

  1. Verify removing a single item from the cart.
  2. Verify removing all items from the cart using a "Clear Cart" option.
  3. Verify the cart is empty after removing all items.
  4. Verify undo functionality (if available) after removing an item.

3. Updating Cart Quantity

  1. Verify increasing the quantity of an item in the cart.
  2. Verify decreasing the quantity of an item in the cart.
  3. Verify quantity update reflects accurately in the total price.
  4. Verify setting the quantity to zero removes the item from the cart.
  5. Verify error message when exceeding available stock during quantity update.

4. Cart Calculations

  1. Verify the cart displays the correct subtotal for each item (price Ã— quantity).
  2. Verify the cart calculates and displays the total price accurately.
  3. Verify the cart calculates discounts or offers correctly.
  4. Verify tax calculations are accurate based on the location.
  5. Verify shipping charges are added to the total price (if applicable).

5. Cart Persistence

  1. Verify that items remain in the cart after the user logs out and logs back in.
  2. Verify cart persistence across different devices (e.g., mobile and desktop).
  3. Verify the cart is cleared after a successful checkout.
  4. Verify session-based cart behavior for guest users (cart data should not persist after the session ends).

6. Cart Usability

  1. Verify cart icon displays the correct item count.
  2. Verify cart displays an appropriate message when empty (e.g., "Your cart is empty").
  3. Verify the product details in the cart (e.g., name, image, price, and quantity) match the selected items.
  4. Verify users can navigate to the product details page from the cart.
  5. Verify the cart summary page displays correctly on different screen sizes.

7. Coupon and Discount Codes

  1. Verify applying valid coupon codes updates the total price.
  2. Verify invalid or expired coupon codes display appropriate error messages.
  3. Verify applying multiple coupons simultaneously (if allowed).
  4. Verify removing applied coupon codes updates the total price.

8. Checkout Flow Integration

  1. Verify the "Proceed to Checkout" button navigates to the checkout page.
  2. Verify users cannot proceed to checkout with an empty cart.
  3. Verify items in the cart reflect correctly on the checkout page.

9. Performance and Security

  1. Verify cart functionality under high traffic conditions.
  2. Verify cart items are secure and cannot be tampered with using external tools.
  3. Verify session timeouts do not result in unauthorized access to the cart.

10. Edge Cases

  1. Verify adding an item to the cart after its price changes.
  2. Verify the cart handles rapid add/remove operations without errors.
  3. Verify cart functionality when the network connection is lost and restored.
  4. Verify behavior when an item in the cart goes out of stock before checkout.
checkout processes

1. General Checkout Flow

  1. Verify that users can proceed to the checkout page from the cart with items in the cart.
  2. Verify users cannot proceed to checkout with an empty cart.
  3. Verify the checkout page is accessible only after user authentication (if applicable).
  4. Verify guest users can access the checkout page (if guest checkout is allowed).

2. Address Management

  1. Verify the user can add a new shipping address during checkout.
  2. Verify selecting a previously saved address for the order.
  3. Verify the ability to edit an existing address during checkout.
  4. Verify mandatory fields in the address form display appropriate validation errors if left blank.
  5. Verify different billing and shipping addresses can be selected (if allowed).
  6. Verify the checkout flow when no address is saved.

3. Payment Options

  1. Verify all available payment methods (e.g., credit card, PayPal, COD) are displayed on the payment page.
  2. Verify successful payment with valid credit/debit card details.
  3. Verify error message for invalid card details (e.g., incorrect CVV, expired card).
  4. Verify that applying a payment method like PayPal redirects to its respective page.
  5. Verify functionality of "Cash on Delivery" (COD) payment option (if available).
  6. Verify saving a card for future payments (if supported).
  7. Verify the payment method selection persists on the review page.

4. Order Summary

  1. Verify the order summary displays all products with correct quantities, prices, and applied discounts.
  2. Verify shipping charges and taxes are calculated correctly.
  3. Verify the final total price matches the item prices, taxes, shipping charges, and discounts.
  4. Verify users can modify the cart from the checkout page.
  5. Verify order summary updates dynamically when cart items or quantities are changed.

5. Discounts and Coupons

  1. Verify the application of valid discount or coupon codes.
  2. Verify error messages for invalid, expired, or inapplicable coupon codes.
  3. Verify removing an applied coupon code reflects updated prices.
  4. Verify combining multiple discounts, if supported (e.g., coupon + store discount).

6. Shipping Options

  1. Verify available shipping methods and their costs display based on the address entered.
  2. Verify estimated delivery dates are displayed correctly.
  3. Verify the user can select a preferred shipping method.
  4. Verify free shipping is applied when conditions (e.g., order amount) are met.

7. Confirmation Page

  1. Verify the order confirmation page displays after successful checkout.
  2. Verify order confirmation email or SMS is sent to the user.
  3. Verify the order confirmation page shows a unique order ID.
  4. Verify the order status is updated in the "My Orders" section after successful payment.

8. Error Handling

  1. Verify the user cannot proceed to checkout with out-of-stock items in the cart.
  2. Verify appropriate error messages if the payment gateway is down.
  3. Verify handling of failed payments (e.g., timeout, insufficient funds).
  4. Verify the checkout page does not crash when large data sets (e.g., many items) are processed.
  5. Verify users are redirected back to the checkout page after a failed payment attempt.

9. Edge Cases

  1. Verify behavior when the product price changes after it is added to the cart but before checkout.
  2. Verify checkout behavior when an item in the cart becomes unavailable before placing the order.
  3. Verify network disconnection during the checkout process and the ability to resume later.
  4. Verify checkout performance under high traffic conditions.
  5. Verify checkout flow for digital/downloadable products without requiring shipping information.
  6. Verify checkout flow for mixed carts (digital + physical products).

10. Security

  1. Verify the checkout process is secured using HTTPS.
  2. Verify sensitive information (e.g., card details) is masked and not stored without user consent.
  3. Verify session expiration logs out the user and clears sensitive data from the checkout page.
  4. Verify protection against duplicate order submissions (e.g., refreshing the confirmation page).
order management

1. Viewing Order History

  1. Verify that users can access their order history from their account/profile page.
  2. Verify that users can view all past orders with order ID, date, status, and total amount.
  3. Verify that users can click on an individual order to see detailed information (items, prices, shipping, etc.).
  4. Verify that users can filter orders by status (e.g., pending, shipped, completed, canceled).
  5. Verify that users can search for past orders using filters (e.g., by order ID, date range, or product name).

2. Order Details

  1. Verify that the order details page displays correct product names, quantities, prices, and shipping address.
  2. Verify that the shipping method, taxes, and any applied discounts are shown clearly on the order details page.
  3. Verify the correct status (e.g., "Pending", "Shipped", "Delivered", "Canceled") is displayed for each order.
  4. Verify that the estimated delivery date is displayed (if applicable).
  5. Verify that the "Track Order" or "Tracking Number" link (if available) directs to the correct external tracking page.

3. Order Status Updates

  1. Verify that the order status changes correctly from "Pending" to "Shipped" when the item is dispatched.
  2. Verify that the status updates to "Delivered" when the order is successfully delivered.
  3. Verify that the order status updates correctly to "Canceled" when the order is canceled by the customer or admin.
  4. Verify that order statuses are updated on the order details page in real-time.

4. Cancelling Orders

  1. Verify that users can cancel orders before shipment if the option is available.
  2. Verify that a confirmation prompt appears when the user attempts to cancel an order.
  3. Verify that canceled orders are moved to the "Canceled" status and are removed from the "Active Orders" section.
  4. Verify that any applicable refunds are processed and updated in the "Payment History" or "Refunds" section.
  5. Verify that users receive a cancellation email/SMS confirmation after canceling an order.
  6. Verify that users are unable to cancel orders after they have been marked as shipped or delivered (if business rules prevent it).

5. Modifying Orders

  1. Verify that users can modify orders (e.g., changing quantity, adding/removing products) before the order is shipped.
  2. Verify that changes to the order update the total price accordingly.
  3. Verify that the user receives a confirmation notification after modifying the order.
  4. Verify that users are not able to modify orders once they are in "Shipped" or "Delivered" status.

6. Return and Refund Process

  1. Verify that users can request a return for an item after delivery (within the return policy window).
  2. Verify that the return process includes selecting the reason for return (e.g., "Defective," "Wrong item," "Changed mind").
  3. Verify that users can track the return status (e.g., "Return Requested," "Return Approved," "Return Completed").
  4. Verify that the refund is processed after a return is approved and the item is received back (refund method and timeline).
  5. Verify that the refunded amount is reflected correctly in the payment history and user account.
  6. Verify that users can track the status of their refund (if supported by the system).

7. Order Invoices and Receipts

  1. Verify that users can view and download an invoice for any past order.
  2. Verify that the invoice includes itemized details such as product names, quantities, prices, shipping charges, taxes, and total amount.
  3. Verify that the invoice contains order number, date, customer information, and payment method.
  4. Verify that users receive an invoice via email upon order completion.

8. Order Tracking

  1. Verify that users can track their order status from the order history page.
  2. Verify that a tracking number is provided for shipped orders, linking to the courier's tracking page.
  3. Verify that tracking information is updated when the status changes (e.g., "Out for Delivery").
  4. Verify that users are notified when the status of their order changes (e.g., order dispatched, out for delivery).
  5. Verify that the system correctly displays "Delivered" once the item has been delivered.

9. Order Notifications

  1. Verify that the user receives an order confirmation email/SMS after placing an order.
  2. Verify that users receive a shipment notification with tracking details after the order is dispatched.
  3. Verify that users receive delivery notifications once their order is delivered.
  4. Verify that users receive cancellation or refund emails when applicable.

10. Multi-Item Orders

  1. Verify that users can view individual statuses of each item in a multi-item order (e.g., shipped, pending, out of stock).
  2. Verify that users are able to track the status of each item in the order separately if one item is delayed.
  3. Verify that users can cancel or return individual items from a multi-item order (if allowed).
  4. Verify that partial refunds are processed correctly when only part of the order is returned.

11. Order History Search and Filter

  1. Verify that users can search their order history by various criteria (e.g., order ID, product name, date range, order status).
  2. Verify that users can filter their order history by status (e.g., "Pending," "Shipped," "Delivered," "Canceled").
  3. Verify that users can view detailed information for orders within the search results.

12. Security

  1. Verify that order information is displayed securely (HTTPS) and cannot be tampered with.
  2. Verify that users cannot view orders that do not belong to them (through authorization checks).
  3. Verify that sensitive order details (e.g., payment method) are masked or hidden for security.

13. Edge Cases

  1. Verify that orders with special conditions (e.g., pre-order, backorder) are handled correctly.
  2. Verify the system can handle multiple simultaneous updates to order status (e.g., bulk order processing).
  3. Verify the handling of orders with multiple currencies (if applicable).
  4. Verify the handling of orders placed during a system outage or under heavy load conditions.

14. Integration Testing

  1. Verify that the order management system integrates correctly with inventory management to reflect stock levels after order placement.
  2. Verify that order information correctly updates in the payment gateway or processor.
  3. Verify that returns and refunds are synchronized across the order management and payment systems.
