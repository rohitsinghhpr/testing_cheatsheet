Authentication

Definition: Authentication is the process of verifying the identity of a user, device, or system. It ensures that the entity trying to access the system is who they claim to be.

Example:

  • Scenario: A user attempts to log into a website by entering a username and password.
  • Authentication Process: The system checks if the username and password match those stored in the database. If they match, the user is authenticated, and access is granted.
  • Example in Action: Logging into an email account using a username and password or using two-factor authentication (e.g., receiving a code on your phone).
  • Question it answers: "Are you who you say you are?"

Authorization

Definition: Authorization is the process of granting or denying access to specific resources or actions after the userâ€™s identity has been authenticated. It determines what an authenticated user is allowed to do within the system.

Example:

  • Scenario: After logging into a banking application, a user attempts to access their account balance.
  • Authorization Process: The system checks whether the authenticated user has permission to view the account balance. If the user has sufficient permissions, the system grants access. If not, the user may be denied access with an error message like "Insufficient permissions."
  • Example in Action: A regular user may have access to their own account information, while an administrator may have access to all user accounts and settings.
  • Question it answers: "What are you allowed to do?"
