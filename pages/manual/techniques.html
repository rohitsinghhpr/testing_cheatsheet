Equivalence Partitioning

  • Description: Divides input data into partitions of equivalent values, where test cases are designed to cover each partition at least once.

  • Goal: Reduces the number of test cases while covering the entire input domain.

  • Example: If a field accepts values from 1–100, equivalence partitions might include:

    • Valid partition: 1–100

    • Invalid partitions: values < 1 and values > 100

Boundary Value Analysis (BVA)

  • Description: Focuses on values at the boundaries of equivalence partitions, as defects often occur at these edges.

  • Goal: To detect errors at boundaries, which are more error-prone.

  • Example: For an input range of 1–100, BVA tests would include values like 0, 1, 100, and 101.

Decision Table Testing

  • Description: Represents combinations of inputs and their associated outcomes in a table format, allowing testers to cover all possible conditions.

  • Goal: To verify different combinations of input conditions and ensure correct outcomes.

  • Example: For a login form with conditions such as correct username and correct password, a decision table would list outcomes for each possible combination of conditions (e.g., correct username + incorrect password, both correct, etc.).

State Transition Testing

  • Description: Tests how the system behaves in various states and transitions between them, especially useful for systems with workflows or state-dependent behavior.

  • Goal: To ensure the system transitions correctly from one state to another based on specific inputs.

  • Example: In a banking application, a state transition test could verify how the system moves from “Account Locked” to “Active” after specific actions like entering the correct security answer.

Use Case Testing

  • Description: Focuses on real-world scenarios as described in use cases, helping to verify that the system functions as intended for specific user interactions.

  • Goal: To ensure the software meets the business and user requirements outlined in use cases.

  • Example: For an e-commerce site, a use case test might cover the process of a user browsing products, adding items to the cart, and completing a purchase.
