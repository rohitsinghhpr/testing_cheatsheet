Water Fall Model

Waterfall Model {linear model}

  1. Requirement analysis (documentation)
  2. System design (design documentation)
  3. Implementation (coding)
  4. Testing
  5. Deployment (install the software in customer environment)
  6. Maintenance

Advantages of waterfall model

  • Quality of product will be good.

  • Since requirement changes are not allowed,changes of finding bugs will be less

  • Initial investment is less since the testers are hired at testing phase.

  • prefered for small projects where requirements are freeze.

    • Disadvantages of waterfall model

  • Requirements changes are not allowed.

  • If there is a defect in requirements that will be continued in later phase.

  • Total investment in more because time taking for rework on defect is time consuming which leads to high investment.

  • Testing will be start after coding.

Spiral Model

Spiral Model

  1. Spiral model is iterative model.
  2. Spiral model overcome drawbacks of waterfall model.
  3. We follow spiral model whenever there is dependency over modules.
  4. In every cycle new software will be released to customer
  5. Software will be released in multiple version. So it is also called version control model.

Advantages of spiral model

  • Testing is done every cycle, before going to next cycle.
  • Customer will get to use software for every module.
  • Requirements changes are allowed after every cycle before going to the next cycle.

Disadvantages of spiral model

  • Requirements changes are not allowed between the cycle.
  • Every cycle of spiral modal looks like a waterfall model.
  • There is no testing in requirements and designing phase.
