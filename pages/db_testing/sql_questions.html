What is SQL?
SQL (Structured Query Language) is a standard programming language used to manage and manipulate relational databases. It allows users to create, retrieve, update, and delete data from database tables.
What are the different types of SQL commands?
SQL commands are categorized into:
  • DDL (Data Definition Language): CREATE, ALTER, DROP, TRUNCATE
  • DML (Data Manipulation Language): SELECT, INSERT, UPDATE, DELETE
  • DCL (Data Control Language): GRANT, REVOKE
  • TCL (Transaction Control Language): COMMIT, ROLLBACK, SAVEPOINT
What is a primary key?
A primary key is a column (or set of columns) in a table that uniquely identifies each row in that table. Primary keys must contain unique values, and null values are not allowed.
What is the difference between WHERE and HAVING clauses?
The WHERE clause is used to filter rows before grouping, while the HAVING clause is used to filter groups after the GROUP BY operation. For example: 

                    SELECT department, COUNT(*)
                    FROM employees
                    WHERE salary > 50000
                    GROUP BY department
                    HAVING COUNT(*) > 5;
What is normalization?
Normalization is the process of organizing database tables to reduce redundancy and improve data integrity. It involves dividing large tables into smaller ones and defining relationships between them. Common forms of normalization include 1NF, 2NF, 3NF, and BCNF.
What is a JOIN in SQL?
A JOIN is used to combine rows from two or more tables based on a related column between them. Types of JOINs include:
  • INNER JOIN: Returns records with matching values in both tables.
  • LEFT JOIN: Returns all records from the left table and matching records from the right table.
  • RIGHT JOIN: Returns all records from the right table and matching records from the left table.
  • FULL OUTER JOIN: Returns all records when there is a match in either table.
What is an index in SQL?
An index is a database object that improves the speed of data retrieval operations on a table. It creates an entry for each value, enabling faster searches. However, indexes can slow down insert and update operations.
What are constraints in SQL?
Constraints are rules enforced on data in tables to ensure accuracy and integrity. Common constraints include:
  • NOT NULL: Ensures a column cannot have a NULL value.
  • UNIQUE: Ensures all values in a column are unique.
  • PRIMARY KEY: Combines NOT NULL and UNIQUE constraints.
  • FOREIGN KEY: Ensures data consistency between related tables.
  • CHECK: Ensures values meet a specific condition.
  • DEFAULT: Assigns a default value to a column.
What is the difference between DELETE and TRUNCATE?
  • DELETE: Removes specific rows from a table based on a condition. It is a DML command and can be rolled back.
  • TRUNCATE: Removes all rows from a table without logging individual row deletions. It is a DDL command and cannot be rolled back.
What is a subquery?
A subquery is a query nested within another query. It is used to perform operations that depend on the result of the outer query. Example: 

                    SELECT name
                    FROM employees
                    WHERE department_id = (SELECT id FROM departments WHERE name = 'Sales');
What is a FOREIGN KEY?
A FOREIGN KEY is a column (or set of columns) in a table that establishes a link between data in two tables. It ensures referential integrity by enforcing that a value in the column must match a value in the referenced column of another table. 

                CREATE TABLE orders (
                    order_id INT PRIMARY KEY,
                    customer_id INT,
                    FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
                );
What are aggregate functions in SQL?
Aggregate functions perform calculations on multiple rows of data and return a single value. Common aggregate functions include:
  • COUNT(): Counts the number of rows.
  • SUM(): Calculates the total of a numeric column.
  • AVG(): Calculates the average of a numeric column.
  • MAX(): Returns the maximum value in a column.
  • MIN(): Returns the minimum value in a column.
Example: 

                SELECT department_id, COUNT(*), AVG(salary)
                FROM employees
                GROUP BY department_id;
What is a view in SQL?
A view is a virtual table based on the result of an SQL query. It allows users to simplify complex queries and provide security by restricting access to specific data. Views do not store data themselves. 

                CREATE VIEW high_salary_employees AS
                SELECT name, department, salary
                FROM employees
                WHERE salary > 100000;
                
                SELECT * FROM high_salary_employees;
What is the difference between UNION and UNION ALL?
  • UNION: Combines the results of two queries and removes duplicate rows.
  • UNION ALL: Combines the results of two queries and includes duplicate rows.
Example: 

                SELECT name FROM employees_department1
                UNION
                SELECT name FROM employees_department2;
        
                SELECT name FROM employees_department1
                UNION ALL
                SELECT name FROM employees_department2;
What is a stored procedure?
A stored procedure is a precompiled set of one or more SQL statements that can be executed as a single unit. Stored procedures help improve performance and maintain code reusability. 

                CREATE PROCEDURE GetEmployeeCount()
                BEGIN
                    SELECT COUNT(*) AS employee_count FROM employees;
                END;
        
                CALL GetEmployeeCount();
What is the difference between a clustered and a non-clustered index?
  • Clustered Index: Determines the physical order of data in a table. Each table can have only one clustered index.
  • Non-Clustered Index: Creates a separate structure for the index, allowing multiple non-clustered indexes per table.
Example: 

                -- Clustered index
                CREATE TABLE employees (
                    id INT PRIMARY KEY CLUSTERED,
                    name VARCHAR(100)
                );
        
                -- Non-clustered index
                CREATE NONCLUSTERED INDEX idx_employee_name ON employees(name);
What is a trigger in SQL?
A trigger is a special type of stored procedure that automatically executes in response to certain events on a table, such as INSERT, UPDATE, or DELETE. 

                CREATE TRIGGER trg_after_insert
                AFTER INSERT ON employees
                FOR EACH ROW
                BEGIN
                    INSERT INTO audit_log(action, log_date)
                    VALUES ('New employee added', NOW());
                END;
What is the difference between CHAR and VARCHAR?
  • CHAR: Fixed-length string data type. It always reserves the specified space, padding with spaces if necessary.
  • VARCHAR: Variable-length string data type. It uses only the space required to store the actual data.
Example: 

                CREATE TABLE example (
                    fixed_length CHAR(10),
                    variable_length VARCHAR(10)
                );
        
                INSERT INTO example VALUES ('abc', 'abc');
What is a CTE (Common Table Expression)?
A Common Table Expression (CTE) is a temporary result set defined within the execution scope of a single SELECT, INSERT, UPDATE, or DELETE statement. It improves readability and simplifies complex queries. 

                WITH SalesCTE AS (
                    SELECT employee_id, SUM(sales) AS total_sales
                    FROM sales
                    GROUP BY employee_id
                )
                SELECT employee_id, total_sales
                FROM SalesCTE
                WHERE total_sales > 5000;
What is a transaction in SQL?
A transaction is a sequence of one or more SQL statements executed as a single unit of work. Transactions follow the ACID properties (Atomicity, Consistency, Isolation, Durability) to ensure reliable processing. 

                BEGIN TRANSACTION;
        
                UPDATE accounts SET balance = balance - 100 WHERE account_id = 1;
                UPDATE accounts SET balance = balance + 100 WHERE account_id = 2;
        
                COMMIT;
                -- Use ROLLBACK instead of COMMIT to undo changes if necessary
What is the difference between RANK(), DENSE_RANK(), and ROW_NUMBER()?
These are window functions used to rank rows within a partition:
  • RANK(): Assigns a rank to each row with gaps for ties.
  • DENSE_RANK(): Assigns a rank without gaps for ties.
  • ROW_NUMBER(): Assigns a unique number to each row, even for ties.
Example: 

                SELECT name, department, salary,
                       RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS rank,
                       DENSE_RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS dense_rank,
                       ROW_NUMBER() OVER (PARTITION BY department ORDER BY salary DESC) AS row_number
                FROM employees;
What is the difference between INNER JOIN and OUTER JOIN?
  • INNER JOIN: Returns rows where there is a match in both tables.
  • OUTER JOIN: Returns all rows from one or both tables, even if there is no match. Types include LEFT, RIGHT, and FULL OUTER JOIN.
Example: 

                -- INNER JOIN
                SELECT e.name, d.department_name
                FROM employees e
                INNER JOIN departments d ON e.department_id = d.department_id;
        
                -- LEFT OUTER JOIN
                SELECT e.name, d.department_name
                FROM employees e
                LEFT JOIN departments d ON e.department_id = d.department_id;
What is the purpose of the GROUP BY clause?
The GROUP BY clause groups rows with the same values into summary rows. It is commonly used with aggregate functions like COUNT(), SUM(), AVG(), MAX(), and MIN(). 

                SELECT department, COUNT(*) AS employee_count, AVG(salary) AS avg_salary
                FROM employees
                GROUP BY department;
What is a self join?
A self join is a regular join where a table is joined with itself. It is useful for comparing rows within the same table. 

                SELECT a.employee_id AS emp1, b.employee_id AS emp2, a.manager_id
                FROM employees a
                INNER JOIN employees b ON a.manager_id = b.employee_id;
What is the difference between COALESCE() and ISNULL()?
  • COALESCE(): Returns the first non-NULL value from a list of expressions. It supports multiple arguments.
  • ISNULL(): Replaces NULL with a specified replacement value. It supports only two arguments and is specific to some databases like SQL Server.
Example: 

                SELECT COALESCE(NULL, NULL, 'Fallback Value') AS result;
                SELECT ISNULL(NULL, 'Fallback Value') AS result; -- SQL Server specific
What is the purpose of the CASE statement in SQL?
The CASE statement provides conditional logic within a query. It allows you to implement IF-THEN-ELSE logic. 

                SELECT name, salary,
                       CASE
                           WHEN salary > 100000 THEN 'High'
                           WHEN salary BETWEEN 50000 AND 100000 THEN 'Medium'
                           ELSE 'Low'
                       END AS salary_category
                FROM employees;
What is the purpose of the EXISTS clause?
The EXISTS clause is used to test for the existence of rows in a subquery. It returns TRUE if the subquery returns one or more rows. 

                SELECT name
                FROM employees e
                WHERE EXISTS (
                    SELECT 1
                    FROM departments d
                    WHERE e.department_id = d.department_id AND d.department_name = 'Sales'
                );
What is the difference between DELETE and DROP?
  • DELETE: Removes specific rows from a table. It is a DML command and can be rolled back.
  • DROP: Deletes an entire table, including its structure. It is a DDL command and cannot be rolled back.
Example: 

                -- DELETE
                DELETE FROM employees WHERE department_id = 10;
        
                -- DROP
                DROP TABLE employees;
What is the purpose of indexing in SQL?
Indexing is used to speed up data retrieval by creating a data structure that allows quick lookups. However, it may slow down write operations like INSERT, UPDATE, and DELETE due to additional maintenance. 

                CREATE INDEX idx_employee_name ON employees(name);
