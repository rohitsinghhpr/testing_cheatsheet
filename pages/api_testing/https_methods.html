What are the HTTP methods used in API requests?
HTTP methods define the actions that can be performed on resources in an API. The primary methods are:
  • GET: Retrieves data from the server. It is a safe and idempotent method, meaning it does not alter data.
  • POST: Sends data to the server to create or update a resource. It is not idempotent, as calling it multiple times can result in different outcomes.
  • PUT: Replaces a resource with new data. It is idempotent, meaning that calling it multiple times with the same data will have the same effect.
  • PATCH: Partially updates a resource. Like PUT, it modifies an existing resource but only updates the specified fields.
  • DELETE: Deletes a resource from the server.
  • OPTIONS: Returns the HTTP methods that are allowed for a specific resource.
  • HEAD: Similar to GET, but it only retrieves the headers, not the body of the response. It's often used to check for the presence of a resource.
What is the GET method in HTTP?
The GET method is used to request data from a specified resource. It is the most common HTTP method used in APIs to retrieve information.
  • Characteristics: It is a safe and idempotent method, meaning it does not modify the resource and can be safely repeated without causing side effects.
  • Usage: Commonly used to retrieve data such as user profiles, product details, and search results.
  • Example: GET /api/v1/users/123 retrieves information about user with ID 123.
Example: 

                GET /api/v1/products HTTP/1.1
                Host: example.com
What is the POST method in HTTP?
The POST method is used to send data to the server to create or update a resource. Unlike GET, POST is not idempotent, meaning that multiple identical requests can result in different outcomes.
  • Characteristics: It is typically used to create resources or submit forms.
  • Usage: Commonly used for actions like user login, creating a new product, or submitting an order.
  • Example: POST /api/v1/users creates a new user.
Example: 

                POST /api/v1/orders HTTP/1.1
                Host: example.com
                Content-Type: application/json
        
                {
                    "product_id": 1,
                    "quantity": 2
                }
What is the PUT method in HTTP?
The PUT method is used to update a resource or create a new resource if it doesn't exist. It replaces the entire resource with the new data sent in the request body.
  • Characteristics: PUT is idempotent, meaning that making the same request multiple times will always produce the same result.
  • Usage: It is used when the client wants to completely replace a resource with new data.
  • Example: PUT /api/v1/users/123 replaces the data of user 123 with new data provided in the request body.
Example: 

                PUT /api/v1/products/1 HTTP/1.1
                Host: example.com
                Content-Type: application/json
        
                {
                    "name": "Updated Product",
                    "price": 20.99
                }
What is the PATCH method in HTTP?
The PATCH method is used to partially update a resource. Unlike PUT, which replaces the entire resource, PATCH only updates the fields provided in the request.
  • Characteristics: PATCH is not idempotent by default, meaning repeated requests with the same data may have different results if the data is not handled correctly.
  • Usage: It is commonly used when updating a single field of a resource, such as changing a user's email address or updating the status of an order.
  • Example: PATCH /api/v1/users/123 updates the user with ID 123 with new data in the request body.
Example: 

                PATCH /api/v1/users/123 HTTP/1.1
                Host: example.com
                Content-Type: application/json
        
                {
                    "email": "newemail@example.com"
                }
What is the DELETE method in HTTP?
The DELETE method is used to delete a resource on the server. It removes the specified resource, and if successful, it generally returns a status code indicating that the resource was deleted.
  • Characteristics: It is idempotent, meaning that calling DELETE multiple times on the same resource will have the same effect (the resource will remain deleted).
  • Usage: Commonly used to delete a user, product, or any other resource from a system.
  • Example: DELETE /api/v1/users/123 deletes the user with ID 123.
Example: 

                DELETE /api/v1/products/1 HTTP/1.1
                Host: example.com
What is the OPTIONS method in HTTP?
The OPTIONS method is used to describe the communication options for the target resource. It allows the client to determine which HTTP methods are supported by a specific resource.
  • Characteristics: It does not require a request body and typically returns an HTTP response header listing supported methods (e.g., GET, POST, PUT, DELETE).
  • Usage: Often used for CORS (Cross-Origin Resource Sharing) requests and to discover allowed methods for a specific endpoint.
  • Example: OPTIONS /api/v1/products returns the allowed HTTP methods for the products resource.
Example: 

                OPTIONS /api/v1/products HTTP/1.1
                Host: example.com
What is the HEAD method in HTTP?
The HEAD method is used to retrieve the headers of a resource, but it does not return the body. It is often used to check for the existence of a resource or get metadata (such as the length of the content or last-modified date).
  • Characteristics: It is similar to GET, but it only retrieves the headers of the response, not the resource data.
  • Usage: Commonly used to check if a resource exists without downloading it or to obtain metadata about a resource.
  • Example: HEAD /api/v1/products/123 retrieves only the headers of the product with ID 123.
Example: 

                HEAD /api/v1/products/123 HTTP/1.1
                Host: example.com
What are idempotent and safe HTTP methods?
  • Idempotent Methods: An HTTP method is idempotent if making multiple identical requests has the same effect as making a single request. For example, PUT and DELETE methods are idempotent because repeating the same request will not have additional effects.
  • Safe Methods: A safe method is one that does not alter the resource. GET, HEAD, OPTIONS, and TRACE are considered safe because they only retrieve data without changing it.
