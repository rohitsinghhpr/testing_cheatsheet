What is Postman?

Definition: Postman is a popular API development and testing tool that allows developers to send requests to APIs, inspect responses, and automate testing of web services.

Primary Use: Postman is primarily used to test RESTful APIs, inspect headers, responses, and automate API testing workflows. It is commonly used for debugging, development, and manual testing of APIs.

Key Benefits:

  • Easy-to-use interface for making API requests and inspecting responses.
  • Support for a wide range of HTTP methods like GET, POST, PUT, DELETE, PATCH, etc.
  • Ability to write and execute automated tests directly within the app.
  • Organize requests into collections and automate API testing workflows.
Postman Features
  • Request Builder: Create and send requests to your APIs with various methods (GET, POST, PUT, DELETE) and parameters.
  • Response Viewer: View API responses, including status codes, headers, and response bodies in different formats (JSON, HTML, XML, etc.).
  • Environment Variables: Define environment variables to store data (like base URLs, authentication tokens) and use them across requests.
  • Automated Testing: Write tests in JavaScript to validate API responses and automate repetitive testing tasks.
  • Collection Runner: Run a series of API requests in sequence to simulate a workflow or test an API endpoint's behavior under different scenarios.
  • Collaboration: Postman allows team members to collaborate on API testing by sharing collections, environments, and test results.

Postman Request Types
  • GET: Retrieve data from the server. Commonly used for fetching resources like user details, posts, etc.
  • POST: Send data to the server. Often used for creating new resources or triggering actions.
  • PUT: Update existing data on the server. Used for full updates of resources (e.g., changing a user's profile).
  • PATCH: Perform partial updates on existing resources. Typically used to update specific fields in a resource.
  • DELETE: Delete a resource from the server, such as removing a user or a post.
Creating and Sending Requests in Postman

Steps to Create a Request:

  1. Select the HTTP method (GET, POST, PUT, DELETE, etc.) from the dropdown.
  2. Enter the API endpoint URL (e.g., https://api.example.com/users).
  3. For methods like POST, PUT, or PATCH, provide the request body in the Body tab (choose raw, form-data, or JSON format as required).
  4. Optionally, add headers such as Content-Type, Authorization, or any custom headers required by the API.
  5. Click "Send" to send the request to the server and view the response.
Postman Environment and Variables

Environment Variables: Postman allows the use of environments to store variables like URLs, authentication tokens, and other data that may change depending on the environment (e.g., development, staging, production).

Defining Variables: You can define variables at the environment or global level. Environment variables are specific to a set of requests, while global variables can be accessed across all collections and environments.

Examples:

  • Base URL: {{base_url}}/api/v1/users
  • Authorization Token: {{auth_token}}

Switching Environments: You can easily switch between different environments by selecting them from the environment dropdown at the top right corner of the Postman window.

Postman Collections

What is a Collection? A collection in Postman is a group of related requests that are organized together. It is used to store and share your API tests and requests with your team.

Creating a Collection: You can create a new collection by clicking on the "New" button and selecting "Collection." Once created, you can add requests to it by clicking on the collection name and selecting "Add Request."

Organizing Requests: Requests in a collection can be organized into folders for better management, especially when dealing with large projects.

Sharing Collections: Collections can be shared with team members by exporting them or using Postman Workspaces to collaborate in real-time.

Postman Tests and Automation

Writing Tests in Postman: You can write automated tests for your APIs using JavaScript. These tests are run after each request is sent to verify the response against expected conditions.

Example Test:

        pm.test("Status code is 200", function () {
            pm.response.to.have.status(200);
        });

Common Assertions:

  • pm.response.to.have.status(code); - Verify response status code.
  • pm.response.to.have.jsonBody(key); - Verify JSON response body contains a specific key.
  • pm.response.to.have.header(name); - Check if a specific header is present in the response.
  • pm.response.to.have.body(value); - Verify if the response body contains a specific value.

Running Tests Automatically: Postman can run collections of tests automatically using the Collection Runner or Newman (the command-line version of Postman).

Postman Collection Runner

What is the Collection Runner? The Collection Runner allows you to run multiple requests in a collection in sequence. It can also be used to execute the same request with different data, such as through CSV or JSON files.

Running a Collection: To run a collection, click on the "Runner" button in the top right of the Postman window, select the collection, and click "Start Run."

Using Data Files: You can use external data files (CSV or JSON) to parameterize your requests during a run. Postman will automatically run the requests with different sets of data.

Viewing Results: After the run, you can view the results, including pass/fail status, response times, and details of each request in the collection.

Postman Monitors

What is a Monitor? A Postman Monitor is a scheduled service that runs collections at specified intervals to test your APIs continuously, ensuring they are working as expected.

Setting Up a Monitor: To create a monitor, navigate to the "Monitor" tab in your collection and set the desired frequency (e.g., hourly, daily). Postman will automatically execute the collection and send notifications on failures.

Use Cases: Monitors are commonly used for:

  • API uptime monitoring
  • Performance testing
  • Regression testing on deployed APIs

Postman Integrations

CI/CD Integrations: Postman can be integrated with CI/CD pipelines using tools like Jenkins. Postman collections can be run in Jenkins jobs using Newman to validate APIs automatically during build/deployment processes.

Other Integrations: Postman also integrates with tools like GitHub, Slack, and Jira for better workflow management, team collaboration, and issue tracking.

Postman Best Practices
  • Organize Requests: Use collections and folders to organize your requests logically (e.g., by feature or endpoint).
  • Use Environment Variables: Manage dynamic values like URLs and tokens using environment variables to avoid hardcoding them.
  • Write Tests: Automate tests to validate API responses and ensure consistent behavior across different environments.
  • Document Your APIs: Use Postman's documentation feature to generate and share API documentation with your team.
  • Version Control: Use version control for your Postman collections (via GitHub or Postman workspaces) to keep track of changes and updates.
Conclusion

Postman is an essential tool for API development, testing, and automation. Its rich features like environment variables, automated tests, and integrations with CI/CD tools help developers and testers streamline their API testing workflows.
