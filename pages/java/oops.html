Classes and Objects

Definition:

A class in Java is a blueprint for creating objects, containing fields (attributes) and methods to define behavior. An object is an instance of a class that holds real values of the fields defined in the class.

Components of a Class:

  • Fields (Attributes): Variables that hold the data or properties of an object.
  • Methods: Functions that define the behavior or actions of the object.
  • Constructors: Special methods to initialize new objects of a class.

Advantages of Using Classes and Objects:

  • Encapsulation: Combines data and methods in a single unit.
  • Reusability: Classes can be reused across programs to create multiple objects with shared structure and behavior.

Example:


    class Car {
        String color;
        String model;
    
        void drive() {
            System.out.println("Driving " + model);
        }
    }
    
    public class Main {
        public static void main(String[] args) {
            Car myCar = new Car();
            myCar.color = "Red";
            myCar.model = "Tesla Model S";
            myCar.drive();
        }
    }

The Car class is used as a blueprint to create an object, myCar, with specific properties and behavior.

Encapsulation

Definition:

Encapsulation is the process of bundling data (fields) and methods that operate on the data into a single unit or class. In Java, encapsulation is primarily achieved through access modifiers and the use of getter and setter methods.

Advantages of Encapsulation:

  • Data Hiding: It restricts direct access to certain components, enhancing security and control over data access.
  • Improved Code Maintenance: By using getter and setter methods, we can modify fields without affecting other parts of the code.
  • Increased Flexibility: Changes in encapsulated code do not require other classes to modify their implementation.

Example:


    class BankAccount {
        private double balance;
    
        public double getBalance() {
            return balance;
        }
    
        public void deposit(double amount) {
            if (amount > 0) {
                balance += amount;
            }
        }
    
        public void withdraw(double amount) {
            if (amount > 0 && balance >= amount) {
                balance -= amount;
            }
        }
    }

Here, the balance field is private, and access to it is provided only through getter and setter methods, ensuring controlled access.

Abstraction

Definition:

Abstraction is the process of hiding complex implementation details and exposing only essential features of an object. In Java, abstraction can be achieved using abstract classes and interfaces.

Benefits of Abstraction:

  • Reduces Complexity: By focusing only on relevant aspects, abstraction makes it easier to understand the system.
  • Improves Code Maintainability: Separating interface from implementation allows changes without affecting other code parts.
  • Encourages Reusability: Abstracted code can be reused in different contexts and by various classes.

Example:


    abstract class Shape {
        abstract void draw();
    }
    
    class Circle extends Shape {
        void draw() {
            System.out.println("Drawing a Circle");
        }
    }
    
    class Square extends Shape {
        void draw() {
            System.out.println("Drawing a Square");
        }
    }

The Shape class is abstract, meaning it defines the draw method but leaves its implementation to derived classes.

Inheritance

Definition:

Inheritance allows a class (subclass) to inherit properties and methods from another class (superclass), promoting code reuse and establishing a hierarchical relationship.

Types of Inheritance in Java:

  • Single Inheritance: A subclass inherits from one superclass.
  • Multilevel Inheritance: A class is derived from a subclass, which is derived from another superclass.

Advantages:

  • Code Reusability: Common functionality in the superclass can be reused in multiple subclasses.
  • Polymorphism: Allows subclasses to provide specific implementations of inherited methods.

Example:


    class Animal {
        void eat() {
            System.out.println("Eating...");
        }
    }
    
    class Dog extends Animal {
        void bark() {
            System.out.println("Barking...");
        }
    }

Here, Dog inherits the eat method from Animal, enabling code reuse.

Polymorphism

Definition:

Polymorphism is the ability of an object to take many forms. It allows methods to perform differently based on the object calling them. In Java, polymorphism is primarily achieved through method overriding and overloading.

Types of Polymorphism:

  • Compile-Time Polymorphism (Method Overloading): Multiple methods have the same name but differ in parameters.
  • Run-Time Polymorphism (Method Overriding): A subclass provides a specific implementation of a method that is already defined in its superclass.

Benefits:

  • Increased Flexibility: Allows different behavior for derived classes without altering the superclass.
  • Code Simplification: Overloading and overriding reduce code duplication by using common method names.

Example of Method Overloading:


    class Printer {
        void print(int number) {
            System.out.println(number);
        }
    
        void print(String text) {
            System.out.println(text);
        }
    }

Example of Method Overriding:


    class Animal {
        void sound() {
            System.out.println("Animal sound");
        }
    }
    
    class Dog extends Animal {
        void sound() {
            System.out.println("Bark");
        }
    }

The sound method is overridden in the Dog class, demonstrating runtime polymorphism.
