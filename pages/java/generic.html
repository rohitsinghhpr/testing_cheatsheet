Generics in Java

Generics allow you to write classes, interfaces, and methods with type parameters. They enable the creation of reusable and type-safe code. With generics, you can work with different data types without sacrificing type safety.

Key Points:

  • Type Safety: Generics ensure that you use the correct data type at compile time, reducing the risk of ClassCastException at runtime.
  • Code Reusability: You can write code that works with any object type.
  • Syntax:
    • For a class: Class<T> where T is a placeholder for a data type.
    • For methods: public <T> void method(T value)
    • For collections: List<String> or List<Integer> instead of List.

Example:

// Generic class
class Box<T> {
private T value;

public void setValue(T value) {
this.value = value;
}

public T getValue() {
return value;
}
}

// Using the generic class
Box<Integer> intBox = new Box<>();
intBox.setValue(10);
System.out.println(intBox.getValue());  // Outputs: 10

Box<String> strBox = new Box<>();
strBox.setValue("Hello");
System.out.println(strBox.getValue());  // Outputs: Hello
