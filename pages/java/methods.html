String Mehtods

Here’s an explanation of commonly used Java String methods along with examples for each:

1. length()

  • Description: Returns the number of characters in the string.
  • Example:
    String str = "Hello";
int length = str.length(); // Returns 5

2. charAt(int index)

  • Description: Returns the character at the specified index.
  • Example:
    String str = "Hello";
char ch = str.charAt(1); // Returns 'e'

3. substring(int start, int end)

  • Description: Returns a new string that is a substring from start index to end index (exclusive).
  • Example:
    String str = "Hello, World";
String sub = str.substring(7, 12); // Returns "World"

4. toUpperCase()

  • Description: Converts all characters in the string to uppercase.
  • Example:
    String str = "hello";
String upper = str.toUpperCase(); // Returns "HELLO"

5. toLowerCase()

  • Description: Converts all characters in the string to lowercase.
  • Example:
    String str = "HELLO";
String lower = str.toLowerCase(); // Returns "hello"

6. trim()

  • Description: Removes leading and trailing whitespace from the string.
  • Example:
    String str = "   Hello   ";
String trimmed = str.trim(); // Returns "Hello"

7. startsWith(String prefix)

  • Description: Checks if the string starts with the specified prefix.
  • Example:
    String str = "Java programming";
boolean result = str.startsWith("Java"); // Returns true

8. endsWith(String suffix)

  • Description: Checks if the string ends with the specified suffix.
  • Example:
    String str = "Java programming";
boolean result = str.endsWith("ing"); // Returns true

9. contains(CharSequence sequence)

  • Description: Checks if the string contains the specified sequence.
  • Example:
    String str = "Hello, Java";
boolean result = str.contains("Java"); // Returns true

10. indexOf(int ch)

  • Description: Returns the index of the first occurrence of the specified character.
  • Example:
    String str = "banana";
int index = str.indexOf('a'); // Returns 1

11. lastIndexOf(int ch)

  • Description: Returns the index of the last occurrence of the specified character.
  • Example:
    String str = "banana";
int index = str.lastIndexOf('a'); // Returns 5

12. replace(CharSequence target, CharSequence replacement)

  • Description: Replaces all occurrences of the target sequence with the replacement sequence.
  • Example:
    String str = "Java is fun";
String replaced = str.replace("fun", "powerful"); // Returns "Java is powerful"

13. concat(String str)

  • Description: Concatenates the specified string to the end of the current string.
  • Example:
    String str1 = "Hello, ";
String str2 = "World!";
String result = str1.concat(str2); // Returns "Hello, World!"

14. isEmpty()

  • Description: Checks if the string is empty (has zero length).
  • Example:
    String str = "";
boolean result = str.isEmpty(); // Returns true

15. format(String format, Object... args)

  • Description: Formats the string using the specified format and arguments.
  • Example:
    String formatted = String.format("Hello, %s!", "World"); // Returns "Hello, World!"

16. split(String regex)

  • Description: Splits the string based on the specified delimiter and returns an array of substrings.
  • Example:
    String str = "apple,banana,orange";
String[] fruits = str.split(","); // Returns ["apple", "banana", "orange"]

17. equals(Object obj)

  • Description: Compares the content of the string to another object to check for equality.
  • Example:
    String str1 = "Java";
String str2 = "Java";
boolean isEqual = str1.equals(str2); // Returns true

18. compareTo(String anotherString)

  • Description: Compares two strings lexicographically.
  • Example:
    String str1 = "Apple";
String str2 = "Banana";
int comparison = str1.compareTo(str2); // Returns negative value as "Apple" is lexicographically less than "Banana"

19. intern()

  • Description: Returns a canonical representation of the string.
  • Example:
    String str1 = new String("Hello").intern();
String str2 = "Hello";
boolean result = (str1 == str2); // Returns true because both refer to the same string pool instance

20. matches(String regex)

  • Description: Checks if the string matches the specified regular expression pattern.
  • Example:
    String str = "hello123";
boolean matches = str.matches("\\w+"); // Returns true (matches a word with alphanumeric characters)

21. valueOf(Object obj)

  • Description: Converts the specified object to a string representation.
  • Example:
    int num = 10;
String str = String.valueOf(num); // Returns "10"
Array Class Methods

Here’s an explanation of the commonly used methods in the Java Arrays class, with examples:

1. toString(int[] array)

  • Description: Converts an array to a string representation.
  • Example:
    int[] numbers = {1, 2, 3, 4};
String arrayString = Arrays.toString(numbers); // Returns "[1, 2, 3, 4]"

2. sort(int[] array)

  • Description: Sorts the array in ascending order. It modifies the array directly.
  • Example:
    int[] numbers = {4, 2, 3, 1};
Arrays.sort(numbers); // numbers becomes [1, 2, 3, 4]

3. binarySearch(int[] array, int key)

  • Description: Searches for a specified element in a sorted array using the binary search algorithm. Returns the index of the element if found; otherwise, returns a negative insertion point.
  • Example:
    int[] numbers = {1, 2, 3, 4, 5};
int index = Arrays.binarySearch(numbers, 3); // Returns 2

4. copyOf(int[] original, int newLength)

  • Description: Creates a copy of the specified array with a new length.
  • Example:
    int[] numbers = {1, 2, 3};
int[] copy = Arrays.copyOf(numbers, 5); // Returns [1, 2, 3, 0, 0]

5. copyOfRange(int[] original, int from, int to)

  • Description: Creates a copy of the specified range from an array.
  • Example:
    int[] numbers = {1, 2, 3, 4, 5};
int[] rangeCopy = Arrays.copyOfRange(numbers, 1, 4); // Returns [2, 3, 4]

6. fill(int[] array, int value)

  • Description: Fills the entire array with the specified value.
  • Example:
    int[] numbers = new int[5];
Arrays.fill(numbers, 9); // Returns [9, 9, 9, 9, 9]

7. equals(int[] a, int[] b)

  • Description: Checks if two arrays are equal by comparing each element in order.
  • Example:
    int[] a = {1, 2, 3};
int[] b = {1, 2, 3};
boolean isEqual = Arrays.equals(a, b); // Returns true

8. deepEquals(Object[] a, Object[] b)

  • Description: Checks if two arrays are equal deeply, including nested arrays.
  • Example:
    Integer[][] a = {{1, 2}, {3, 4}};
Integer[][] b = {{1, 2}, {3, 4}};
boolean isEqual = Arrays.deepEquals(a, b); // Returns true

9. hashCode(int[] array)

  • Description: Returns a hash code based on the array’s contents.
  • Example:
    int[] numbers = {1, 2, 3};
int hashCode = Arrays.hashCode(numbers); // Returns hash code value

10. deepHashCode(Object[] array)

  • Description: Returns a deep hash code for an array, which considers nested arrays.
  • Example:
    Integer[][] array = {{1, 2}, {3, 4}};
int hashCode = Arrays.deepHashCode(array); // Returns deep hash code

11. asList(T... a)

  • Description: Converts an array to a List backed by the array.
  • Example:
    String[] words = {"apple", "banana", "cherry"};
List<String> list = Arrays.asList(words); // Returns [apple, banana, cherry]

12. deepToString(Object[] array)

  • Description: Converts a multi-dimensional array to a string.
  • Example:
    Integer[][] array = {{1, 2}, {3, 4}};
String arrayString = Arrays.deepToString(array); // Returns "[[1, 2], [3, 4]]"

13. toString(Object[] array)

  • Description: Converts a single-dimensional array to a string.
  • Example:
    Integer[] array = {1, 2, 3, 4};
String arrayString = Arrays.toString(array); // Returns "[1, 2, 3, 4]"

14. parallelSort(int[] array)

  • Description: Sorts the array in parallel for performance on large datasets.
  • Example:
    int[] numbers = {4, 2, 3, 1};
Arrays.parallelSort(numbers); // numbers becomes [1, 2, 3, 4]

15. setAll(int[] array, IntUnaryOperator generator)

  • Description: Sets each element of the array using a generator function.
  • Example:
    int[] numbers = new int[5];
Arrays.setAll(numbers, i -> i * 2); // Returns [0, 2, 4, 6, 8]

16. stream(int[] array)

  • Description: Returns a sequential stream for the array, allowing for stream operations.
  • Example:
    int[] numbers = {1, 2, 3, 4, 5};
int sum = Arrays.stream(numbers).sum(); // Returns 15

17. mismatch(int[] a, int[] b)

  • Description: Finds the first index where two arrays differ. Returns -1 if they are identical.
  • Example:
    int[] a = {1, 2, 3};
int[] b = {1, 2, 4};
int mismatchIndex = Arrays.mismatch(a, b); // Returns 2

18. compare(int[] a, int[] b)

  • Description: Compares two arrays lexicographically.
  • Example:
    int[] a = {1, 2, 3};
int[] b = {1, 2, 4};
int comparison = Arrays.compare(a, b); // Returns negative because a < b

19. compareUnsigned(int[] a, int[] b)

  • Description: Compares two arrays lexicographically treating values as unsigned.
  • Example:
    int[] a = {1, 2, 3};
int[] b = {1, 2, 4};
int comparison = Arrays.compareUnsigned(a, b); // Returns negative because a < b

The Arrays class in Java provides a powerful set of methods for array manipulation, from basic operations like sorting and searching to advanced parallel sorting and array transformations. These methods make array processing more efficient and code more readable.

Math Methods

Explanation of Java Math Methods with Examples:

  1. abs(int value) – Returns the absolute value of the given integer.

    • Example: abs(-10) returns 10.

  2. max(int a, int b) – Returns the maximum of two integers.

    • Example: max(5, 10) returns 10.

  3. min(int a, int b) – Returns the minimum of two integers.

    • Example: min(5, 10) returns 5.

  4. sqrt(double value) – Returns the square root of the given double.

    • Example: sqrt(25) returns 5.0.

  5. pow(double base, double exponent) – Returns the result of raising the base to the power of exponent.

    • Example: pow(2, 3) returns 8.0.

  6. log(double value) – Returns the natural logarithm (base e) of a double.

    • Example: log(2.718) returns approximately 1.0.

  7. random() – Returns a random double between 0.0 (inclusive) and 1.0 (exclusive).

    • Example: random() might return 0.54321.

  8. round(double value) – Rounds the double to the nearest integer.

    • Example: round(4.6) returns 5.

  9. ceil(double value) – Returns the smallest integer greater than or equal to the input double.

    • Example: ceil(4.3) returns 5.0.

  10. floor(double value) – Returns the largest integer less than or equal to the input double.

    • Example: floor(4.8) returns 4.0.
Date Methods

The Date class in Java provides a variety of methods to work with dates and times. Here’s a breakdown of the key methods along with examples:

1. Date() Constructor

  • Description: Creates a Date object initialized to the current date and time.
  • Example:
    Date currentDate = new Date(); // Current date and time

2. Date(long date) Constructor

  • Description: Creates a Date object initialized to the specified number of milliseconds since January 1, 1970, 00:00:00 GMT.
  • Example:
    Date specificDate = new Date(1633046400000L); // Date for specified milliseconds

3. getTime()

  • Description: Returns the number of milliseconds since January 1, 1970, 00:00:00 GMT.
  • Example:
    Date date = new Date();
long milliseconds = date.getTime(); // Milliseconds since epoch

4. setTime(long time)

  • Description: Sets the Date object to represent a point in time that is time milliseconds after January 1, 1970, 00:00:00 GMT.
  • Example:
    Date date = new Date();
date.setTime(1633046400000L); // Set date to specific milliseconds since epoch

5. before(Date when)

  • Description: Checks if the date is before the specified date.
  • Example:
    Date date1 = new Date(1633046400000L);
Date date2 = new Date(1633132800000L);
boolean isBefore = date1.before(date2); // Returns true if date1 is before date2

6. after(Date when)

  • Description: Checks if the date is after the specified date.
  • Example:
    Date date1 = new Date(1633046400000L);
Date date2 = new Date(1633132800000L);
boolean isAfter = date2.after(date1); // Returns true if date2 is after date1

7. equals(Object obj)

  • Description: Checks if two dates are exactly equal.
  • Example:
    Date date1 = new Date(1633046400000L);
Date date2 = new Date(1633046400000L);
boolean isEqual = date1.equals(date2); // Returns true if date1 and date2 are the same

8. compareTo(Date anotherDate)

  • Description: Compares two dates, returning a negative, zero, or positive value based on whether the date is before, equal to, or after the specified date.
  • Example:
    Date date1 = new Date(1633046400000L);
Date date2 = new Date(1633132800000L);
int comparison = date1.compareTo(date2); // Returns negative if date1 is before date2

9. toString()

  • Description: Returns a string representation of the date in the format: E MMM dd HH:mm:ss z yyyy.
  • Example:
    Date date = new Date();
String dateString = date.toString(); // Example output: "Wed Oct 01 00:00:00 GMT 2021"

10. toInstant()

  • Description: Converts the Date to an Instant object, which represents a specific point in time in UTC.
  • Example:
    Date date = new Date();
Instant instant = date.toInstant(); // Converts Date to Instant

11. from(Instant instant) (Static Method)

  • Description: Creates a Date object from an Instant.
  • Example:
    Instant instant = Instant.now();
Date dateFromInstant = Date.from(instant); // Converts Instant to Date

The Date class provides a basic structure for working with dates and times, though it has largely been replaced by the java.time package in Java 8 and later for more complex date-time manipulation. The Date class is still commonly used for simple date and time storage and for interacting with legacy code.

Vector Class Methods

The Vector class is part of the Java Collections Framework and provides a dynamic array-like data structure that can grow or shrink as needed. Unlike ArrayList, Vector is synchronized, which means it is thread-safe. However, due to its synchronization, Vector tends to be slower in performance compared to ArrayList in single-threaded scenarios.

1. add(E e)

  • Description: Appends the specified element to the end of the vector.
  • Example:
    Vector<String> vector = new Vector<>();

vector.add("Banana");

2. add(int index, E element)

  • Description: Inserts the specified element at the specified position in the vector.
  • Example:
    Vector<String> vector = new Vector<>();
vector.add("Apple");
vector.add("Banana");
vector.add(1, "Cherry");  // Adds "Cherry" at index 1

3. get(int index)

  • Description: Returns the element at the specified index.
  • Example:
    Vector<String> vector = new Vector<>();
vector.add("Apple");
vector.add("Banana");
String element = vector.get(0); // Returns "Apple"

4. set(int index, E element)

  • Description: Replaces the element at the specified position in the vector with the specified element.
  • Example:
    Vector<String> vector = new Vector<>();
vector.add("Apple");
vector.add("Banana");
vector.set(1, "Cherry");  // Replaces "Banana" with "Cherry"

5. remove(int index)

  • Description: Removes the element at the specified position in the vector.
  • Example:
    Vector<String> vector = new Vector<>();
vector.add("Apple");
vector.add("Banana");
vector.remove(0);  // Removes "Apple"

6. remove(Object o)

  • Description: Removes the first occurrence of the specified element from the vector.
  • Example:
    Vector<String> vector = new Vector<>();
vector.add("Apple");

  • Description: Returns the number of elements in the vector.

8. isEmpty()

Stack

Stack Class Methods

The Stack class in Java extends the Vector class and represents a last-in, first-out (LIFO) stack of objects. It provides methods to push and pop elements from the stack.

1. push(E item)

  • Description: Pushes an item onto the top of the stack.
  • Example:
    Stack<String> stack = new Stack<>();
stack.push("Apple");  // Adds "Apple" to the stack
stack.push("Banana"); // Adds "Banana" to the stack

2. pop()

  • Description: Removes and returns the top element from the stack. Throws EmptyStackException if the stack is empty.
  • Example:
    Stack<String> stack = new Stack<>();
stack.push("Apple");
String topElement = stack.pop();  // Removes "Apple" and returns it

3. peek()

  • Description: Returns the top element of the stack without removing it. Throws EmptyStackException if the stack is empty.
  • Example:
    Stack<String> stack = new Stack<>();
stack.push("Apple");
String topElement = stack.peek();  // Returns "Apple" without removing it

4. isEmpty()

  • Description: Checks if the stack is empty.
  • Example:
    Stack<String> stack = new Stack<>();
boolean empty = stack.isEmpty();  // Returns true

5. search(Object o)

  • Description: Returns the 1-based position of the object from the top of the stack. Returns -1 if the element is not found.
  • Example:
    Stack<String> stack = new Stack<>();
stack.push("Apple");
stack.push("Banana");
int position = stack.search("Banana");  // Returns 1 (since "Banana" is at the top)

6. size()

  • Description: Returns the number of elements in the stack.
  • Example:
    Stack<String> stack = new Stack<>();
stack.push("Apple");
stack.push("Banana");
int size = stack.size();  // Returns 2

Key Characteristics of Vector and Stack

  • Thread Safety: Both Vector and Stack are synchronized, making them thread-safe. However, this synchronization can lead to performance overhead in single-threaded environments.
  • Dynamic Growth: The Vector class grows dynamically as needed when elements are added, unlike arrays that have a fixed size.
  • Stack Specifics: The Stack class is a subclass of Vector that provides LIFO (Last-In, First-Out) behavior, with methods like push, pop, and peek to mimic stack operations.

In modern Java programming, ArrayList and Deque (or LinkedList) are often preferred over Vector and Stack for better performance in non-concurrent environments. However, Vector and Stack are still commonly used in legacy systems.

Key Characteristics of HashSet:

  • Uniqueness: A HashSet does not allow duplicate elements. If you try to add an element that is already present, it will not be added again.
  • Unordered: The elements in a HashSet are not stored in any particular order. The order may change when you iterate over the set.
  • Backed by HashMap: Internally, HashSet is backed by a HashMap, where the elements are stored as keys, and a dummy value is used for each key.
  • Performance: Most operations (add, remove, contains) are O(1) on average due to the underlying hash table.

The HashSet class is part of the Java Collections Framework and is commonly used when you need to store unique elements in an unordered collection. It provides fast lookups and ensures no duplicate values.
