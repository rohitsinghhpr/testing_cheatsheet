WebDriver Methods

WebDriver Methods

These methods are used to control the browser and manage the WebDriver instance:

  1. get(url): Opens the specified URL in the browser.
  2. getCurrentUrl(): Returns the current URL of the browser.
  3. getTitle(): Returns the title of the current page.
  4. findElement(By by): Finds the first WebElement using the given method.
  5. findElements(By by): Finds all WebElements using the given method.
  6. getPageSource(): Returns the source code of the current page.
  7. close(): Closes the current browser window.
  8. quit(): Closes all browser windows and ends the WebDriver session.
  9. getWindowHandles(): Returns a set of window handles for all open browser windows.
  10. getWindowHandle(): Returns the window handle of the current window.
  11. switchTo(): Switches to a different frame, window, or alert.
    • switchTo().frame()
    • switchTo().window()
    • switchTo().alert()
  12. navigate(): Allows navigation to and from web pages.
    • navigate().to(url)
    • navigate().back()
    • navigate().forward()
    • navigate().refresh()
  13. manage(): Provides various options to manage the browser.

    • manage().timeouts()

      • implicitlyWait(Duration duration): Sets the amount of time the driver should wait when searching for an element if it is not immediately present; use this to handle situations where elements take time to appear.
        • Scenario: When elements on a page load asynchronously and you want to ensure that WebDriver waits long enough to find all elements without throwing an exception immediately.
      • pageLoadTimeout(Duration duration): Sets the maximum amount of time to wait for a page to load before throwing an error; use this to handle slow-loading pages.
        • Scenario: When navigating to a webpage that might take a longer time to load due to heavy content or slow server response.
      • scriptTimeout(Duration duration): Sets the amount of time to wait for an asynchronous script to finish execution before throwing an error; use this for operations that involve executing JavaScript with a delay.
        • Scenario: When executing JavaScript code via executeAsyncScript that involves asynchronous operations, such as AJAX calls, which might take some time to complete.

    • manage().window()

      Sure, here are concise explanations of each sub-method of manage().window():

      • maximize(): Maximizes the browser window to fill the screen.
      • minimize(): Minimizes the browser window to the taskbar.
      • fullscreen(): Switches the browser window to full-screen mode.
      • setPosition(Point targetPosition): Sets the browser window's position to specified x and y coordinates.
      • setSize(Dimension targetSize): Sets the browser window's size to the specified width and height.
      • getPosition(): Returns the current position of the browser window as a Point object.
      • getSize(): Returns the current size of the browser window as a Dimension object.

    • manage().cookies()

      Sure, here are concise explanations of each sub-method of manage().cookies():

      • addCookie(Cookie cookie): Adds a specified cookie to the current domain.
      • deleteCookie(Cookie cookie): Deletes a specified cookie from the current domain.
      • deleteCookieNamed(String name): Deletes a cookie by its name.
      • deleteAllCookies(): Deletes all cookies from the current domain.
      • getCookieNamed(String name): Returns the cookie with the specified name.
      • getCookies(): Returns a set of all cookies for the current domain.
  14. executeScript(script, *args): Executes JavaScript in the context of the current window or frame.
  15. executeAsyncScript(script, *args): Executes asynchronous JavaScript in the context of the current window or frame.

Implicit Wait

Implicit wait instructs WebDriver to wait for a certain amount of time when trying to find an element if it is not immediately available. It sets a default wait time for the entire duration of the WebDriver session.

                                
                            // Set implicit wait
                            driver.manage().timeouts().implicitlyWait(10, TimeUnit.SECONDS);
                            // Open a website
                            driver.get("<http://example.com>");
                            // Interact with elements
                            // WebDriver will wait up to 10 seconds for elements to be found
                            driver.findElement(By.id("someElementId")).click();
Explict Wait

Explicit wait allows you to specify a certain condition to wait for before proceeding further in the code. It provides more flexibility compared to implicit wait as you can target specific elements and conditions.

Fluent Wait

Fluent wait is a more advanced type of explicit wait that allows you to set the maximum amount of time to wait for a condition, as well as the frequency with which to check the condition. Additionally, you can ignore specific types of exceptions while waiting.

Use fluent wait when you need more control over the waiting behavior, such as polling intervals and exception handling.

Selenium Exception

Selenium WebDriver can throw various exceptions during the execution of automation scripts. Understanding these exceptions and their solutions is crucial for writing robust and reliable test scripts. Here are some common Selenium Java exceptions and their solutions:

  1. NoSuchElementException

    • Description: Thrown when an element could not be found on the web page.

    • Solution: Ensure the locator used to find the element is correct and wait for the element to be present.

      WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));
        WebElement element = wait.until(ExpectedConditions.presenceOfElementLocated(By.id("elementId")));

  2. StaleElementReferenceException

    • Description: Thrown when a reference to an element is no longer valid (e.g., the element has been refreshed or removed from the DOM).

    • Solution: Re-acquire the element reference after the page updates or use explicit waits.

      WebElement element = driver.findElement(By.id("elementId"));
        element.click();
        // Re-locate the element if necessary
        element = driver.findElement(By.id("elementId"));

  3. TimeoutException

    • Description: Thrown when a command does not complete in the specified timeout period.

  4. ElementNotVisibleException

    • Description: Thrown when an element is present in the DOM but not visible.

    • Solution: Ensure the element is visible before interacting with it or use waits to handle visibility.

      WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));
        WebElement element = wait.until(ExpectedConditions.visibilityOfElementLocated(By.id("elementId")));

  5. ElementNotInteractableException

    • Description: Thrown when an element is not interactable (e.g., it is disabled or not visible).

    • Solution: Wait for the element to become interactable or ensure it is in the correct state.

      WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));
        WebElement element = wait.until(ExpectedConditions.elementToBeClickable(By.id("elementId")));

  6. NoAlertPresentException

    • Description: Thrown when switching to an alert but no alert is present.

    • Solution: Ensure an alert is present before switching to it.

      WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));
        wait.until(ExpectedConditions.alertIsPresent());
        Alert alert = driver.switchTo().alert();
        alert.accept();

  7. InvalidSelectorException

    • Description: Thrown when an invalid selector is used.

    • Solution: Verify the correctness of the selector syntax.

      WebElement element = driver.findElement(By.cssSelector("valid-css-selector"));

  8. WebDriverException

    • Description: Generic exception thrown when something goes wrong with the WebDriver.
    • Solution: Investigate the root cause by checking the error message and stack trace. Ensure WebDriver and browser versions are compatible.

  9. MoveTargetOutOfBoundsException

    • Description: Thrown when the target for a mouse action is outside the view.

    • Solution: Scroll the element into view before performing the action.

      WebElement element = driver.findElement(By.id("elementId"));
        ((JavascriptExecutor) driver).executeScript("arguments[0].scrollIntoView(true);", element);

  10. SessionNotFoundException

    • Description: Thrown when the session ID is not found (e.g., when the session is terminated).

    • Solution: Ensure the WebDriver session is properly initialized and not terminated prematurely.

      WebDriver driver = new ChromeDriver();
        driver.get("<https://www.example.com>");
        driver.quit();

  11. UnhandledAlertException

    • Description: Thrown when there is an unhandled alert on the web page.

    • Solution: Handle the alert before performing further actions.

      try {
        Alert alert = driver.switchTo().alert();
        alert.accept();
        } catch (NoAlertPresentException e) {
        // No alert found, proceed with the test
        }

  12. InvalidElementStateException

    • Description: Thrown when an element is in an invalid state for the action being performed (e.g., trying to send keys to a non-input element).
    • Solution: Ensure the element is in the correct state before performing the action.
    WebElement element = driver.findElement(By.id("inputElement"));
        if (element.isEnabled()) {
        element.sendKeys("text");
        }
Page Object Model

Page Object Model -

The Page Object Model (POM) is a design pattern in Selenium for enhancing test maintenance and reducing code duplication. It represents web pages as classes, where each class contains web elements and methods to interact with those elements. When used with TestNG, POM helps organize test cases and manage test execution efficiently.

Key Components of POM

  1. Page Class: Represents a web page and contains web elements and methods to interact with them.

  2. Test Class: Contains the actual test methods, utilizing the page classes to perform actions.

Advantages of POM

  • Improved Readability: Tests become more readable and understandable.

  • Reusability: Methods in page classes can be reused across multiple tests.

  • Maintainability: Changes in the UI need to be updated only in one place, the page class.

Example Structure

  1. Base Class: Contains common setup and teardown methods.

  2. Page Classes: Represent different web pages.

  3. Test Classes: Contain test methods.

Page Factory

Page Factory -

Page Factory is a class in Selenium that supports the Page Object Model (POM) design pattern by providing a way to initialize web elements defined in Page Objects. It makes the code more readable and easier to maintain by automatically initializing web elements at runtime using the @FindBy annotation.

Key Components of Page Factory

  1. Page Class: Represents a web page and contains web elements annotated with @FindBy.
  2. PageFactory Class: Contains the initElements method to initialize the web elements.

Example Structure

  1. Base Class: Contains common setup and teardown methods.
  2. Page Classes: Represent different web pages and use @FindBy for web elements.
  3. Test Classes: Contain test methods that utilize the page classes

Methods in PageFactory

  1. PageFactory.initElements(WebDriver driver, Object page):

    • Initializes the elements of the given page object.
    • Automatically locates the elements defined by the @FindBy annotation. 
    PageFactory.initElements(driver, this);

Annotations in PageFactory

  1. @FindBy:

    • Used to locate elements using different strategies (id, name, className, xpath, etc.).
    @FindBy(id = "username")
    private WebElement usernameField;

  2. @FindBys:

    • Used to locate elements using multiple criteria (AND condition).
    @FindBys({
    @FindBy(className = "class1"),
    @FindBy(className = "class2")
    })
    private WebElement element;

  3. @FindAll:

    • Used to locate elements using multiple criteria (OR condition).
    @FindAll({
    @FindBy(id = "username"),
    @FindBy(name = "username")
    })
    private WebElement usernameField;
