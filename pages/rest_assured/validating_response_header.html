Basic Header Validation
Validating a specific header and its value in the response: import static io.restassured.RestAssured.*; import static org.hamcrest.Matchers.*; public class ApiTest { public static void main(String[] args) { given() .when() .get("https://jsonplaceholder.typicode.com/posts/1") .then() .header("Content-Type", "application/json; charset=utf-8"); } } This example checks that the Content-Type header in the response is application/json; charset=utf-8.
Multiple Header Validation
Validating multiple headers in a single request: 

        import static io.restassured.RestAssured.*;
        import static org.hamcrest.Matchers.*;
        
        public class ApiTest {
            public static void main(String[] args) {
                given()
                    .when()
                    .get("https://jsonplaceholder.typicode.com/posts/1")
                    .then()
                    .header("Content-Type", "application/json; charset=utf-8")
                    .header("Connection", "keep-alive");
            }
        }
This verifies both the Content-Type and Connection headers in the response.
Validating Header with Partial Match
Using a partial match for headers with dynamic or changing values: 

        import static io.restassured.RestAssured.*;
        import static org.hamcrest.Matchers.*;
        
        public class ApiTest {
            public static void main(String[] args) {
                given()
                    .when()
                    .get("https://jsonplaceholder.typicode.com/posts/1")
                    .then()
                    .header("Content-Type", containsString("application/json"));
            }
        }
This checks that the Content-Type header contains application/json, allowing flexibility for additional charset information.
Checking for Presence of a Header
Verifying if a specific header is present in the response without checking its exact value: 

        import static io.restassured.RestAssured.*;
        import static org.hamcrest.Matchers.*;
        
        public class ApiTest {
            public static void main(String[] args) {
                given()
                    .when()
                    .get("https://jsonplaceholder.typicode.com/posts/1")
                    .then()
                    .headers("Content-Type", notNullValue());
            }
        }
This ensures that the Content-Type header is present in the response.
Validating Header with Multiple Possible Values
Checking for multiple acceptable values for a header: 

        import static io.restassured.RestAssured.*;
        import static org.hamcrest.Matchers.*;
        
        public class ApiTest {
            public static void main(String[] args) {
                given()
                    .when()
                    .get("https://jsonplaceholder.typicode.com/posts/1")
                    .then()
                    .header("Connection", anyOf(equalTo("keep-alive"), equalTo("close")));
            }
        }
This example verifies that the Connection header is either keep-alive or close.
