Handling Path Parameters
Path parameters are dynamic parts of the URL. In Rest Assured, they can be set using the pathParam method: 

            import static io.restassured.RestAssured.*;
            
            public class ApiTest {
                public static void main(String[] args) {
                    given()
                        .pathParam("postId", 1)
                        .when()
                        .get("https://jsonplaceholder.typicode.com/posts/{postId}")
                        .then()
                        .statusCode(200);
                }
            }
This example sets postId as a path parameter, making the URL https://jsonplaceholder.typicode.com/posts/1.
Handling Multiple Path Parameters
To pass multiple path parameters, use multiple pathParam calls: 

            import static io.restassured.RestAssured.*;
            
            public class ApiTest {
                public static void main(String[] args) {
                    given()
                        .pathParam("userId", 1)
                        .pathParam("postId", 5)
                        .when()
                        .get("https://jsonplaceholder.typicode.com/users/{userId}/posts/{postId}")
                        .then()
                        .statusCode(200);
                }
            }
This sets userId and postId as path parameters in the URL.
Handling Query Parameters
Query parameters are appended to the URL after a ? and are often used for filtering results. Use queryParam to add them in Rest Assured: 

            import static io.restassured.RestAssured.*;
            
            public class ApiTest {
                public static void main(String[] args) {
                    given()
                        .queryParam("userId", 1)
                        .when()
                        .get("https://jsonplaceholder.typicode.com/posts")
                        .then()
                        .statusCode(200);
                }
            }
This example adds userId=1 as a query parameter, making the request URL https://jsonplaceholder.typicode.com/posts?userId=1.
Handling Multiple Query Parameters
You can set multiple query parameters by chaining multiple queryParam methods: 

            import static io.restassured.RestAssured.*;
            
            public class ApiTest {
                public static void main(String[] args) {
                    given()
                        .queryParam("userId", 1)
                        .queryParam("title", "foo")
                        .when()
                        .get("https://jsonplaceholder.typicode.com/posts")
                        .then()
                        .statusCode(200);
                }
            }
This example creates a URL with multiple query parameters, like https://jsonplaceholder.typicode.com/posts?userId=1&title=foo.
Passing Parameters Using Maps
Both path and query parameters can be passed using a Map, making it easier to handle dynamic values: 

            import static io.restassured.RestAssured.*;
            import java.util.Map;
            import java.util.HashMap;
            
            public class ApiTest {
                public static void main(String[] args) {
                    // Path parameter using a Map
                    Map pathParams = new HashMap<>();
                    pathParams.put("postId", 1);
                    
                    // Query parameters using a Map
                    Map queryParams = new HashMap<>();
                    queryParams.put("userId", 1);
                    queryParams.put("title", "foo");
            
                    given()
                        .pathParams(pathParams)
                        .queryParams(queryParams)
                        .when()
                        .get("https://jsonplaceholder.typicode.com/posts/{postId}")
                        .then()
                        .statusCode(200);
                }
            }
In this example, both path and query parameters are passed in Map objects.
