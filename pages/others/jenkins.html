What is Jenkins?

Definition: Jenkins is an open-source automation tool written in Java that helps in automating various stages of software development, including building, testing, and deploying.

Primary Use: Jenkins is primarily used for Continuous Integration (CI), which ensures that code changes are automatically integrated and tested in the development environment.

Key Benefits:

  • Automates repetitive tasks like builds and tests.
  • Facilitates CI/CD pipelines.
  • Supports distributed workloads across multiple machines.
Jenkins Features for Automation Testing

Integration with Version Control: Jenkins integrates with Git, SVN, and other version control systems to fetch the latest code for building and testing.

Test Automation Integration: Jenkins can execute automated tests (e.g., Selenium, JUnit, TestNG) and report the results in a detailed manner.

Distributed Builds: Jenkins allows running tests on multiple machines simultaneously, improving performance and reducing execution time.

Plugins: Jenkins has a rich ecosystem of plugins, allowing integration with other tools, such as build tools (Maven, Gradle), test frameworks (JUnit, TestNG), and reporting tools.

Continuous Integration/Continuous Deployment (CI/CD): Jenkins helps automate the entire lifecycle from code commits to deployment, ensuring that the software is always in a deployable state.

Key Jenkins Components
  • Jenkins Master: The central server that orchestrates all Jenkins jobs and builds.
  • Jenkins Slave: A remote machine used to offload tasks (builds, tests) from the master server.
  • Build Jobs: A Jenkins job defines the steps to build and test the software. You can configure a job to run tests, compile code, deploy the application, etc.
  • Pipeline: A Jenkins pipeline is a suite of automated processes for building, testing, and deploying code. It can be written using a domain-specific language (DSL) called "Pipeline as Code" in Groovy.
Setting Up Jenkins for Automation Testing

Install Jenkins: Jenkins can be installed using various methods:

  • Using native packages (DEB/RPM)
  • Using Docker
  • Using WAR file (Java application)

Install Plugins: For automation testing, install plugins such as:

  • JUnit Plugin (to integrate JUnit results with Jenkins)
  • TestNG Plugin (to display TestNG results in Jenkins)
  • Maven Plugin (for building and managing dependencies)
  • Selenium Plugin (for integrating Selenium WebDriver tests)
  • Allure Plugin (for generating beautiful reports for test results)
  • Git Plugin (for integrating Jenkins with version control repositories like GitHub, Bitbucket, etc.)

Creating and Running Jenkins Jobs for Automation Testing

Create a Job: 1. Navigate to the Jenkins dashboard. 2. Click on "New Item". 3. Choose "Freestyle project" or "Pipeline" depending on your need. 4. Configure the job to run the automated tests using commands or scripts.

Configure Source Code Management (SCM): Choose Git, SVN, or other SCM systems to link your Jenkins job to the repository. Provide the repository URL and any credentials if necessary.

Configure Build Triggers:

  • Poll SCM: Jenkins can poll your source control system at regular intervals for new changes and trigger the build automatically.
  • GitHub Webhooks: You can configure Jenkins to trigger builds when code is pushed to a repository.

Configure Build Steps:

  • Execute Shell or Windows Batch Commands: Run your test scripts via command-line instructions (e.g., mvn test for Maven).
  • Invoke Gradle/Maven: Use these tools to compile the code and run unit tests.
  • Run Selenium Tests: Configure Jenkins to execute Selenium WebDriver tests by invoking your test scripts.

Pipeline as Code in Jenkins (Jenkinsfile)

Jenkinsfile: A Jenkinsfile is a text file that defines a Jenkins pipeline using a domain-specific language (DSL). It describes the stages involved in the pipeline, such as building, testing, and deploying the application.

Example Jenkinsfile (Declarative Pipeline):

        pipeline {
            agent any
            stages {
                stage('Build') {
                    steps {
                        sh 'mvn clean install'
                    }
                }
                stage('Test') {
                    steps {
                        sh 'mvn test'
                    }
                }
                stage('Deploy') {
                    steps {
                        sh 'mvn deploy'
                    }
                }
            }
        }

Pipeline Types:

  • Declarative Pipelines: Easier to read and manage; most commonly used.
  • Scripted Pipelines: More flexible, allows more complex logic but requires more setup.

Automated Test Execution and Reporting

JUnit/TestNG Results: Jenkins collects the results of your automated tests and displays them in an organized way. This helps testers track the results of each test run and diagnose failures.

Email Notifications: Jenkins can notify the team via email if a test fails, or even send custom notifications with failure details.

Allure Report Plugin: To generate beautiful, detailed, and customizable test reports. Allure integrates with JUnit, TestNG, and other testing frameworks.

Common Jenkins Commands for Automation Testers

Trigger Jenkins job via command line:

        curl -X POST http:///job//build

Trigger Jenkins job with parameters:

        curl -X POST http:///job//buildWithParameters?PARAM1=value1&PARAM2=value2

Check build status:

        curl http:///job//lastBuild/api/json
Best Practices for Jenkins Automation Testers
  • Use Version Control for Jenkinsfiles: Store your Jenkinsfile in the same Git repository as your application code.
  • Parallel Test Execution: To speed up test execution, use parallel pipelines or run tests on different machines (distributed builds).
  • Use Virtual Machines or Docker Containers: Set up Jenkins agents on virtual machines or Docker containers to isolate environments and prevent issues with different test environments.
  • Monitor and Archive Test Results: Archive test reports, logs, and artifacts (like screenshots) for later reference and debugging.
  • Build Fast Feedback Loops: Configure Jenkins jobs to run unit tests, integration tests, and other checks quickly so that testers get immediate feedback on code changes.
Conclusion

Jenkins is a powerful tool that can significantly enhance the efficiency of an automation tester by automating the integration and testing processes. By utilizing Jenkins' capabilities such as continuous integration, automated test execution, and integration with various testing frameworks, testers can ensure faster feedback, more reliable testing, and better collaboration across the development team.
